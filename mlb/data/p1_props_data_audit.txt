SECTION 6 — BUILD ROADMAP
======================================================================

TIER 1 — BUILD FIRST (highest data quality + existing infrastructure):

  1. PITCHER STRIKEOUTS
     Why first:
     - Existing projection engine (modules/props_projections.py)
     - Existing odds pipeline (modules/props_odds.py for DK lines)
     - 100% data coverage (K, IP, BF, pitches all at 0% null)
     - Opponent K% derivable from hitter logs
     - Most liquid prop market — posted early, available on all books
     - Starter-specific (1 projection per game per team = clean)

  2. BATTER TOTAL BASES
     Why second:
     - Existing DK odds pipeline
     - TB derivable (1B + 2×2B + 3×3B + 4×HR)
     - ISO available for power profiling
     - Park factors available
     - Second-most liquid hitter prop market

TIER 2 — BUILD NEXT (good data, needs projection model):

  3. BATTER HITS
     - Clean data (0% null)
     - Simpler outcome than TB (binary per AB)
     - Needs: rolling contact rate model

  4. PITCHER OUTS RECORDED
     - Directly derivable from IP
     - Needs: innings projection model

TIER 3 — BUILD LATER (context-dependent):

  5. BATTER HOME RUNS
     - Low base rate (~10% of games have 1+ HR)
     - Needs large rolling windows
     - Park + weather interaction important

  6. EARNED RUNS ALLOWED
     - Available but volatile per-game metric
     - Park-dependent

  7. RBIs / RUNS SCORED
     - Highly context-dependent (lineup position, runners on base)
     - Build only after Tier 1-2 validated
