SECTION 3 — MODEL LOGIC BREAKDOWN
======================================================================

TWO ENGINES — RULES (current) vs SIMULATION (Phase 9)

RULES ENGINE (modules/projections.py) — Currently active
----------------------------------------------------------------------

  Type: MULTIPLICATIVE
  Base: 4.50 runs per team × 2 teams = 9.0 neutral total

  Per-team formula:
    runs = base × (SP_IP/9 × SP_factor × OFF_factor × PF × weather × ump)
         + base × (BP_IP/9 × BP_factor × OFF_factor × PF × weather × ump)

  Where:
    SP_factor = blended(xFIP, SIERA) / 4.25
    OFF_factor = wRC+ / 100
    PF = park_factor / 100
    weather = wind_adj × temp_factor (capped [0.85, 1.15])
    ump = umpire_runs_factor (~0.97–1.03)
    BP_factor = fatigue_multiplier (~0.95–1.30)

  Total = runs_away + runs_home

  Properties:
    - Multiplicative: factors compound (strong offense + weak pitcher = big effect)
    - No explicit variance modeling
    - No distribution output (single point estimate)
    - F5: caps SP innings at 5, zero bullpen
    - Treats lineups as team-level wRC+ (no individual hitter)

SIMULATION ENGINE (modules/sim_projections.py) — Phase 9 ready
----------------------------------------------------------------------

  Type: RIDGE REGRESSION + MONTE CARLO
  Input: 25 features → Ridge(α=50) → predicted total
  Variance: separate 12-feature Ridge → per-game σ
  Simulation: 50,000 draws from N(predicted_total, σ)

  Output:
    - Point estimate (Ridge prediction, clamped [4.0, 22.0])
    - P(over) and P(under) vs market line
    - 80% confidence interval [10th, 90th percentile]
    - Per-game σ (from variance model)

  Properties:
    - Produces full distribution (not just mean)
    - Models variance explicitly (wind, flyball → wider σ)
    - Umpire σ multiplier (tight zone → lower variance)
    - Still team-level (no individual hitter modeling)
    - F5 = full × 0.56 (static fraction, not modeled separately)

KEY ANSWERS:

  Is the model additive, multiplicative, or simulation-based?
    Rules: MULTIPLICATIVE
    Sim: LINEAR REGRESSION + SIMULATION

  Does it produce a distribution or just a mean?
    Rules: MEAN ONLY
    Sim: FULL DISTRIBUTION (N(μ, σ))

  Does it model variance?
    Rules: NO
    Sim: YES (12-feature variance model)

  Does it distinguish first 5 vs full game?
    Rules: YES (separate F5 calc, zero bullpen)
    Sim: PARTIAL (F5 = full × 0.56 constant, not independent model)

  Does it treat lineups dynamically or statically?
    STATICALLY — team-level wRC+ only. No hitter-level adjustment.

SECTION 6 — WEAKNESSES / GAPS
======================================================================

PLAYER-LEVEL GAPS
----------------------------------------------------------------------
  ✗ No hitter-level modeling
    - Offense is team wRC+ only (PA-weighted season average)
    - No individual batter contributions
    - No awareness of WHO is in the lineup beyond SP

  ✗ No lineup-based adjustments
    - Confirmed lineups are detected (lineup_monitor.py) but not
      used to adjust wRC+ or run projections
    - A team's wRC+ is the same regardless of which 9 hitters start

  ✗ No platoon splits in model
    - Platoon wRC+ (vs LHP/RHP) is FETCHED by offense.py
    - But NOT used in the Ridge model or rules engine
    - Available data is wasted

  ✗ No per-player contribution modeling
    - Cannot estimate "how much does Player X add to the total?"
    - Cannot model usage redistribution when a player is out

PITCHER GAPS
----------------------------------------------------------------------
  ✗ No pitch mix data
    - No fastball %, breaking ball %, or pitch type distribution
    - Cannot model pitcher-specific platoon exposure

  ✗ No times-through-order (TTO) effects
    - TTO flag exists in feature_table but is NOT used in model
    - Pitchers degrade 3rd time through order — not captured

  ✗ No HR susceptibility modeling
    - Park HR factor exists but no pitcher-level HR/FB rate
    - Cannot model flyball pitcher + HR park interaction properly

  ✗ No recent form / streakiness
    - Pitcher stats are season-level (static)
    - No rolling last-5-starts xFIP or last-10-days ERA
    - Cannot detect in-season deterioration or improvement

  ✗ No velocity tracking
    - Velocity decline is a strong fatigue signal — not modeled

BULLPEN GAPS
----------------------------------------------------------------------
  ✓ Bullpen fatigue IS tracked (Phase 8)
  ✓ Closer availability IS tracked
  ✗ No individual reliever modeling
    - Team-level BP xFIP only; no per-reliever quality
  ✗ No leverage-aware usage prediction
    - Cannot model "will the closer pitch tonight?"
  ✗ No bullpen usage sequence prediction
    - Cannot estimate which relievers will face which hitters

ENVIRONMENT GAPS
----------------------------------------------------------------------
  ✓ Weather well-modeled (wind direction + speed + temperature)
  ✓ Park factors included (runs + HR)
  ✓ Umpire modeled (runs_factor + variance multiplier)
  ✗ No humidity / barometric pressure (affects ball carry)
  ✗ No altitude effect beyond Colorado
  ✗ No park-specific pitcher interaction
    - e.g., flyball pitcher in Coors vs groundball pitcher in Coors
    - wind×flyball exists but park×pitcher type does not

STRUCTURAL GAPS
----------------------------------------------------------------------
  ✗ F5 model is not independent
    - F5 = full_game × 0.56 (constant fraction)
    - Should be a separate model with F5-specific features
    - SP dominance is higher in F5 — different weight structure

  ✗ No run distribution modeling
    - Model produces expected total only
    - Sim adds N(μ,σ) but no skew, kurtosis, or bimodality
    - Real baseball scores are discrete and slightly right-skewed

  ✗ No team total modeling
    - Only game total (home + away combined)
    - Cannot bet team totals (e.g., NYY over 4.5)

  ✗ No home/away run split prediction
    - runs_home and runs_away are produced by rules engine
    - But Ridge model predicts combined total only
    - Cannot derive moneyline or run line edges

  ✗ No matchup interaction layer
    - Hitter vs pitcher matchup not modeled
    - Platoon advantage not incorporated
    - Team offensive style vs pitcher style not considered

DATA GAPS
----------------------------------------------------------------------
  ✗ No historical player game logs (hitter-level)
    - Only team wRC+ exists; no individual game-by-game stats
    - Cannot build hitter projections without this

  ✗ No lineup history
    - Cannot train on "who actually batted" in historical games
    - Only game-day confirmed lineups (not archived systematically)

  ✗ No historical opening lines (2022-2023)
    - Opening lines exist only for NBA (just backfilled)
    - MLB opening lines would enable movement analysis

  ✗ Closing lines incomplete for 2022-2023
    - mlb_historical_closing_lines has 3,911 rows (partial)
    - 2024-2025 complete via Phase 7 backfill

  ✗ No pitcher game logs
    - pitch-count history, velocity trends, rest patterns
    - Only season aggregates from FanGraphs

----------------------------------------------------------------------

PHASE M6 — PARALLEL MODEL ARCHITECTURE
----------------------------------------------------------------------
Priority: LOW (until M2-M5 validated)
Depends on: M5

  Model A — Current Production (Ridge, 25 features)
    - Continue as baseline comparison
    - No modifications

  Model B — Lineup-Driven Ridge
    - Replace team wRC+ with lineup-adjusted wRC+
    - Add platoon, SP recent form, TTO features
    - Same Ridge architecture, expanded feature set (~35 features)
    - Backtest: does lineup info improve over team average?

  Model C — Distribution-Based
    - Model run distribution (not just mean)
    - Poisson or negative binomial per team
