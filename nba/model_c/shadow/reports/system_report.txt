NBA MODEL C SHADOW SYSTEM — SETUP REPORT
==========================================
Created: 2026-03-19
Status: OPERATIONAL

SYSTEM OVERVIEW
===============

Fully automated NBA player props shadow loop covering:
- Points, Rebounds, Assists, PRA props
- 8 sportsbooks (DK, FD, BetMGM, Caesars, BetRivers, etc.)
- Automatic RS/PO split detection
- Daily collection, projection, grading, and archiving

COMMANDS
========

  python3 nba/model_c_shadow.py --collect          # Morning: pull lines + project + card
  python3 nba/model_c_shadow.py --refresh           # Midday: re-pull lines (preserves first seen)
  python3 nba/model_c_shadow.py --grade             # Next morning: grade yesterday's plays
  python3 nba/model_c_shadow.py --summary           # Print RS and PO summaries
  python3 nba/model_c_shadow.py --collect --date 2026-03-20

DAILY WORKFLOW
==============

  7:00 AM  →  --grade (grades yesterday's plays from box scores)
  9:00 AM  →  --collect (pull today's lines + project + shadow card)
  5:00 PM  →  --refresh (re-pull pre-tip lines, track movement)

LAUNCHD INTEGRATION
===================

The collect run can be added to the morning 7am job alongside other sports.
The grade step should run before collect to update yesterday's results first.
The refresh can be added to the 5pm refresh job.

Suggested integration in push_results.py or a dedicated script:
  1. Grade yesterday: python3 nba/model_c_shadow.py --grade
  2. Collect today:   python3 nba/model_c_shadow.py --collect

RS / PLAYOFF SPLIT
==================

Detection method:
  1. Check NBA Stats API for playoff game data
  2. Fallback: date >= 2026-04-19 → PO
  3. RS and PO write to separate parquet files
  4. Summaries are independent

When playoffs begin, the system automatically:
  - Detects PO phase
  - Writes to model_c_po_shadow.parquet
  - Keeps RS data untouched
  - Reports PO separately in --summary

PROJECTION ENGINE
=================

Model C v1 uses L10 rolling averages as baseline projection:
  - Points:   pts_L10 (last 10 games average)
  - Rebounds:  reb_L10
  - Assists:   ast_L10
  - PRA:       pra_L10

Confidence tiers by absolute edge:
  - HIGH:   |edge| >= 3.0
  - MEDIUM: |edge| >= 1.5
  - LOW:    |edge| >= 0.5
  - NO_PLAY: |edge| < 0.5 (excluded from shadow card)

Future v2 improvements:
  - Ridge regression on rate × minutes
  - Opponent defensive adjustment
  - Usage redistribution when teammates are OUT
  - Starter vs bench weighting
