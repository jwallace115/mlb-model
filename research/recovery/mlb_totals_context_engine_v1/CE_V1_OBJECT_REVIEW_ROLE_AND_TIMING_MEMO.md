# CE V1 — Object-Review Role and Timing Rules

**Frozen:** 2026-04-14  
**Type:** Role-and-timing guardrail memo

---

## 1. PURPOSE

This memo freezes the correct role of CE V1 after the ESP × SS over-side branch closure. That branch demonstrated the key lesson: descriptive structural leadership does not equal exploitable mechanism. This memo exists to prevent CE V1 from drifting back into a discovery tool. It is not a betting memo.

## 2. WHAT CE V1 IS FOR

- **False-positive detection:** challenging candidate objects against known structural response tendencies
- **Structural footprint characterization:** profiling which CE V1 states a candidate inhabits
- **Regime splitting:** understanding whether a candidate lives in stable or noisy structural territory
- **Residual / baseline-deviation review:** asking whether a candidate's directional claim exceeds the 51.4/41.8 structural baseline
- **Descriptive postmortem:** reviewing why a candidate succeeded or failed after the fact

CE V1 is best used to interrogate a candidate object, not invent one.

## 3. WHAT CE V1 IS NOT FOR

- Not a direct signal generator
- Not a standalone betting engine
- Not proof of market error or edge
- Not a substitute for mechanism development
- Not a substitute for normal PIT-safe testing
- Not a first-step discovery tool

## 4. REQUIRED SEQUENCE OF USE

The correct sequence is:

1. **Identify a candidate mechanism** outside CE V1 — from domain knowledge, market observation, or structural reasoning
2. **Harden the mechanism** — define it precisely, state falsification conditions
3. **Test the mechanism normally** — PIT-safe discovery, validation, OOS with pre-registered thresholds
4. **Only if it survives normal testing**, use CE V1 to review:
   - Structural footprint (which SS/ESP/WPL states does it inhabit?)
   - Contradiction/alignment (is it swimming with or against structural currents?)
   - Residual edge vs structural baseline (does it exceed the 51.4/41.8 baseline?)
   - Regime concentration (is it living in FRAGILE or otherwise noisy states?)
   - False-positive risk (is it just rediscovering generic structural response?)

CE V1 review is **step 4 infrastructure**, not step 1 discovery. Using CE V1 earlier risks turning it back into a mechanism-generation tool — which is exactly what failed in the ESP × SS branch.

## 5. THE DECISION-TIME IDENTITY PROBLEM

- Full-pregame CE V1 state is appropriate for **descriptive and postmortem** use
- Live candidate characterization requires CE V1 state at the **object's actual decision time** (e.g., 7 AM, 10 AM, 12 PM ET)
- That decision-time CE V1 snapshot is **not yet frozen**
- Until it is frozen, live decision-time characterization is **not identity-safe**
- This is an unresolved design problem, not a solved one

The P1B decision-time branch was paused for exactly this reason — temperature and price identity could not be established at candidate decision times without new infrastructure. Future work must keep this caveat visible.

## 6. VALID USES RIGHT NOW

- Descriptive postmortem review of candidate objects
- Branch postmortems (e.g., ESP × SS closure review)
- False-positive challenge questions during object design
- Residualization / baseline inheritance review
- Regime-splitting review
- Structural characterization of already-existing candidates, **if explicitly labeled as descriptive/postmortem** when using full-pregame CE V1 state

## 7. NOT-YET-VALID USES

- Live decision-time filtering using CE V1
- Automated contradiction/alignment labeling in production
- Treating full-pregame CE V1 state as if it were a 7 AM decision-time snapshot
- Using CE V1 as first-step discovery machinery
- Treating descriptive structural states as betting objects

## 8. WHAT FUTURE RESEARCHERS MUST REMEMBER

- Descriptive structure is not the same as exploitable mechanism — the ESP × SS branch proved this
- CE V1 should be compared against the 51.4/41.8 structural baseline, not 50/50
- A candidate must exist and survive normal testing before CE V1 review matters
- Timing semantics must match the object if the work is not explicitly postmortem
- Self-audits that miss execution violations are not trustworthy — the front-loaded and lineup-amplifier executions both had minor process flags
- CE V1 remains valuable even though the ESP × SS mechanism branch failed — it is infrastructure, not alpha

## 9. STATUS

CE V1 is frozen as descriptive/object-review infrastructure. It is not a signal engine, not a substitute for candidate testing, and not yet identity-safe for live decision-time characterization. No live or shadow object behavior has been changed.

This memo is frozen as a role-and-timing guardrail. Future work should reference it before any CE V1 object-review prompt is written.
